The Problem

Slow Transfers & Internet Dependency

Current file-sharing methods often require active internet connections or suffer from extremely slow transfer speeds over Bluetooth/traditional apps.

  • Cloud storage needs data/internet.
  • Large files (5GB+) take hours on basic apps.
  • Privacy risks on public networks.

The Solution: FastShare

A Local Network First application that creates a direct, high-speed tunnel between devices.

  • 100% Offline (No Data/Internet needed).
  • Uses WiFi/LAN at hardware limits.
  • Secure, private, and extremely fast.

Workflow: Sending

1. Select files (even huge 50GB ones).

2. Set an optional password.

3. App starts a local HTTP server.

4. Displays a unique QR code with the server address.

Workflow: Receiving

1. Scan the QR code using the app or any browser.

2. Enter password (if required).

3. Authenticated session starts.

4. Download files at maximum hardware speed.

Handling 50GB+ Files

Stream vs. Load

Most apps crash because they try to load files into RAM. FastShare uses Chunked Binary Streaming.

  • RAM usage stays low (<100MB) regardless of file size.
  • Supports 4GB+ files on FAT32/exFAT systems.
  • Resilient against network fluctuations.
